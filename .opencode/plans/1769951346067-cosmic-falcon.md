# Human-in-the-Loop Implementation Plan

## Overview
Implement a comprehensive "human in the loop" system that allows the AI agent to hand off to human agents when uncertain, provides pause/play controls at conversation and shop levels, and intelligently detects human interventions.

## Current State
- **Database**: Convex with `shops`, `threads`, `messages`, `pending_messages` tables
- **Thread Status**: Currently supports "active", "resolved", "archived"
- **Shop Settings**: Has `autoReplyEnabled` boolean and `businessHours` fields
- **AI Processing**: Messages batched and processed via `processPendingMessages` action
- **UI**: Studio dashboard with thread sidebar (`thread-sidebar.tsx`) and conversation view (`conversation-view.tsx`)
- **No handoff mechanism exists** - marked as TODO in improvements list

## Implementation Phases

### Phase 1: Per-Conversation Pause/Play (Priority: High)

**Database Changes** (`convex/schema.ts`):
```typescript
// Add to threads table:
agentStatus: v.optional(v.union(
  v.literal("active"),      // AI responding normally
  v.literal("paused"),      // Human has taken over
  v.literal("handoff"),     // AI requested handoff
  v.literal("pending_human") // Waiting for human response
)),
agentPausedAt: v.optional(v.number()),
agentPausedBy: v.optional(v.id("users")), // For tracking who paused
lastHumanMessageAt: v.optional(v.number()), // Track human intervention
```

**Backend Changes**:
1. **Update `convex/threads.ts`**:
   - Add `pauseThread` mutation to set agentStatus to "paused"
   - Add `resumeThread` mutation to set agentStatus to "active"
   - Add `requestHandoff` mutation for AI-triggered handoffs
   - Update `getOrCreate` to default agentStatus to "active"

2. **Update `convex/scheduledActions.ts`**:
   - Check thread's `agentStatus` before processing
   - If status is "paused" or "handoff", skip AI processing
   - If status is "pending_human", queue message but don't process
   - Add logic to detect when AI should request handoff (see Phase 3)

3. **Update `convex/messages.ts`**:
   - Add `addHumanMessage` mutation for dashboard-sent messages
   - When human sends message, automatically set thread status to "paused"
   - Update `addMessage` to track `lastHumanMessageAt` when role is not "assistant"

**UI Changes**:
1. **Update `app/studio/[threadId]/page.tsx`**:
   - Add pause/play button in header
   - Show current agent status with visual indicator
   - Add handoff button for manual human takeover

2. **Update `components/studio/thread-list-item.tsx`**:
   - Show pause/handoff status indicator on thread items
   - Visual distinction for paused conversations (e.g., different background, icon)

3. **Create `components/studio/agent-status-controls.tsx`**:
   - Pause/Resume button component
   - Handoff button component
   - Status badge component

### Phase 3: Confidence-Based Handoff (Priority: High)

**System Prompt Updates** (`convex/scheduledActions.ts` - `ISUPPLY_SYSTEM_PROMPT`):
Add new section to system prompt:
```
## When to Handoff to Human

You should request a handoff to a human agent when:
1. You're unsure about a policy or procedure ("I need to confirm that with my team")
2. The customer is upset or frustrated ("Let me get a human to help you with this")
3. The question requires human judgment (custom orders, special requests)
4. You cannot find the information after checking tools
5. The customer explicitly requests a human

**To request handoff:** End your response with ||HANDOFF|| followed by a brief reason.
Example: "I'm not sure about our trade-in policy for that specific device. ||HANDOFF|| Need to verify trade-in eligibility for non-Apple devices"
```

**Backend Changes**:
1. **Update `convex/scheduledActions.ts`**:
   - Parse AI response for `||HANDOFF||` delimiter
   - If found, extract reason and call `requestHandoff` mutation
   - Send handoff notification to human agents (could be in-app notification, Slack, email)
   - Don't send the ||HANDOFF|| portion to the customer

2. **Update `convex/threads.ts`**:
   - Add `requestHandoff` mutation that:
     - Sets thread status to "handoff"
     - Stores handoff reason
     - Creates notification for human agents
     - Optionally sends message to customer: "Let me get someone to help you with that"

**UI Changes**:
1. **Create notification system**:
   - Add `notifications` table to schema
   - Create notification component for dashboard
   - Show handoff requests prominently

2. **Update `app/studio/page.tsx`**:
   - Show handoff requests as priority items
   - Filter/sort threads by handoff status

### Phase 4: Human Response Detection & Auto-Pause (Priority: Medium)

**Architecture Decision**:
The system needs to distinguish between:
- Customer messages (role: "user")
- AI messages (role: "assistant") 
- Human agent messages (need new role or flag)

**Database Changes** (`convex/schema.ts`):
```typescript
// Update messages table role field:
role: v.union(
  v.literal("user"),           // Customer
  v.literal("assistant"),      // AI Agent
  v.literal("human_agent")     // Human agent via dashboard
),

// Add to threads table:
hasHumanIntervention: v.optional(v.boolean()),
lastHumanAgentId: v.optional(v.id("users")),
```

**Backend Changes**:
1. **Create new mutation** `convex/messages.ts` - `sendHumanAgentMessage`:
   - Validates user is authenticated
   - Sends message to customer via Meta API
   - Saves message with role: "human_agent"
   - Automatically sets thread status to "paused"
   - Updates `lastHumanMessageAt` and `hasHumanIntervention`

2. **Update `convex/scheduledActions.ts`**:
   - Check if thread has `hasHumanIntervention` flag
   - If true and last message is from "human_agent", stay paused
   - Resume only when explicitly triggered by human agent

3. **Update webhook handlers** (`app/api/webhooks/messenger/route.ts`, `app/api/webhooks/instagram/route.ts`):
   - When receiving customer message, check if thread is paused
   - If paused due to human intervention, don't trigger AI processing
   - Queue message for when human resumes

**UI Changes**:
1. **Create message composer** `components/studio/message-composer.tsx`:
   - Text input for human agents to respond
   - Send button that calls `sendHumanAgentMessage`
   - Only show when thread is in handoff or paused state

2. **Update `components/studio/conversation-view.tsx`**:
   - Show "human_agent" messages with different styling (e.g., "Agent" label instead of "Customer")
   - Add message composer at bottom when thread is paused
   - Show "Paused - Human Agent Responding" banner

## Testing Strategy

1. **Unit Tests**:
   - Test thread status state machine transitions
   - Test business hours calculation with different timezones
   - Test message role assignment (user/assistant/human_agent)

2. **Integration Tests**:
   - Test AI processing respects pause status
   - Test human message triggers auto-pause
   - Test shop-wide pause affects all threads
   - Test handoff notification delivery

3. **End-to-End Tests**:
   - Full flow: Customer message → AI response → Pause → Human responds → Resume → AI responds
   - Handoff flow: AI requests handoff → Human notified → Human takes over
   - Shop pause: Pause shop → Multiple conversations queued → Resume → All processed

## Migration Plan

1. **Database Migration**:
   - Add new fields as optional (backward compatible)
   - Backfill existing threads with agentStatus = "active"
   - Backfill shops with settings.agentPaused = false

2. **Deployment Order**:
   1. Deploy schema changes
   2. Deploy backend mutations and queries
   3. Deploy UI changes
   4. Update system prompt for handoff detection
   5. Enable for test shop
   6. Rollout to all shops

## Success Metrics

- **Handoff Accuracy**: AI correctly identifies when to handoff (measure via human feedback)
- **Response Time**: Time from human request to human response
- **Auto-pause Rate**: % of human responses that correctly trigger auto-pause
- **Shop Pause Usage**: How often shop-wide pause is used
- **Customer Satisfaction**: No decrease (or increase) in satisfaction scores

## Future Enhancements

1. **AI Handoff Prediction**: Use model confidence scores instead of delimiter parsing
2. **Human Agent Assignment**: Route handoffs to specific agents based on expertise
3. **Escalation Rules**: Auto-escalate if no human responds within X minutes
4. **Handoff Analytics**: Track handoff reasons to improve AI training
5. **Mobile Notifications**: Push notifications for handoff requests
6. **Collaboration**: Multiple human agents can collaborate on a conversation
